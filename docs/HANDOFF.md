## What this is

A **static Hebrew RTL photo-album website** for a family's 1-year trip
(Nepal → India → Vietnam → China → Australia → New Zealand → Thailand).
Photos live in a public Google Drive folder; the site never re-hosts them.

- **Live:** https://hermantrip.tomhe.app  (VPS: `root@tomhe.app`, Caddy + Cloudflare)
- **Stack:** vanilla JS, native ES modules, **no bundler / no build step**.
  Pure logic in `src/lib/` (no DOM/fetch), views in `src/views/` (pure
  HTML-string builders), all DOM/router/state wiring in `src/main.js`.
- **Tests:** `node --test` (zero deps). Currently **524 passing / 0 fail**.
- **Manifest:** `data/manifest.json` (~4.8 MB, 88 raw albums → 87 after
  merge), generated by `scripts/build-manifest.mjs` (rclone), checked in.
- **Methodology:** cr-tdd-ladder — every change = branch → PR → **CR-ist
  subagent review** (`Agent` subagent_type `crist`, rules in `docs/cr-rules.md`)
  → literal merge → `v0.MN` tag → archived dist tarball in `versions/`.

## Deploy (standing instruction from the user)

> "each time you finish a version, upload it." SSH key is saved, empty
> passphrase, no password.

Implemented as tar-over-ssh of a freshly-assembled `dist/`:
```
tar czf - -C dist . | ssh root@tomhe.app 'rm -rf /var/www/hermantrip/* && tar xzf - -C /var/www/hermantrip'
```
**IMPORTANT:** when adding a new `src/lib/*.js` or `src/views/*.js`, add it to
BOTH (a) the dist-copy file list in the deploy command and (b) `sw.js`
`SHELL_FILES`, and **bump `SHELL_CACHE`** (currently `hermantrip-shell-v33`)
so returning visitors get the new code via the SW auto-update.

## Key architecture notes

- **Routing (clean paths, no hash):** `/nepal/nagarkot-bhaktapur`,
  `/nepal/poon-hill-trek/3` (slide idx). `src/lib/router.js` + `ROUTES` in
  `main.js`. Caddy `try_files {path} /index.html` for SPA deep links.
- **Album slugs (M23):** `src/lib/album-slugs.js` maps album id → English
  place slug + aliases. Numeric/unknown URLs → 404. `albumBySlug()` in
  `album-query.js` resolves slug→album (alias → replaceState to canonical).
- **Image proxy:** browser can't hotlink lh3 (ORB); `/img/{id}/{width}` is
  reverse-proxied by Caddy (and `scripts/serve.mjs` in dev) to
  `lh3.../d/{id}=w{width}`. All `<img>` URLs go through `src/lib/image-url.js`.
- **Service worker:** network-first shell, **bypasses** the manifest and
  `/img/`. Auto-update (`src/lib/sw-update.js`) reloads once on a new SW so
  users don't get stuck on stale builds.
- **Map/Globe (`/map`):** Leaflet (lazy via script tag) + Globe.gl (lazy via
  `import('https://esm.sh/globe.gl@2.34.0')`). `src/lib/map-stops.js` is the
  ordered "trip stops" source of truth for BOTH pins and the gradient trail.
- **Timeline (`/timeline`):** all 325 day-shells render up front (empty
  `.tl-photo-strip[data-bucket-index]`), hydrated lazily by an
  IntersectionObserver (0.5 s dwell) + on slider jump. View = `src/views/timeline.js`,
  wiring = `renderTimelineView()` in `main.js`.

## How to work here

1. `git checkout -b mN-slug` (or `fix/slug`). Make the change.
2. Write/extend `*.test.mjs`; capture FAIL-before / PASS-after for the PR body.
3. `node --test` must be 0-fail. **Also run `npx eslint@9` on touched JS** until
   it's wired into the suite.
4. Build `dist/`, bump `sw.js` cache + `SHELL_FILES` if you added a module.
5. `gh pr create` → spawn the CR-ist (`Agent`, `subagent_type: crist`) →
   **wait for APPROVED** → `gh pr merge --merge` → tag `v0.MN` → archive →
   deploy → **verify live in the browser**.
6. After ANY `src/main.js` edit: confirm the byte change actually landed
   (Edits silently failed multiple times this session — `grep` the result).
